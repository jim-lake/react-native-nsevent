# react-native-nsevent Specification

## Overview

`react-native-nsevent` is a React Native macOS TurboModule that exposes low-level window-scoped input events using AppKit `NSEvent` APIs.

The module uses:

- `NSEvent addLocalMonitorForEventsMatchingMask:`
- `std::atomic<int64_t>` mouse delta accumulation
- TurboModule callback (`std::function`) for keyboard/button/scroll events
- TurboModule synchronous method for mouse delta polling

The module intentionally does **not** use:

- CGEventTap
- Accessibility permissions
- Input Monitoring permissions
- Global event interception

---

# Architecture

## JS Layer (TypeScript spec)

The JS layer defines a typed TurboModule interface and retrieves the module via `TurboModuleRegistry`.

### Files

```text
src/
    NativeRNNSEvent.ts   ← TurboModule spec (codegen input)
    index.ts             ← public API re-exports
```

### TurboModule Spec

```typescript
import type { TurboModule } from 'react-native';
import { TurboModuleRegistry } from 'react-native';

export interface Spec extends TurboModule {
  registerEventCallback(callback: ((event: Object) => void) | null): void;
  startCapture(): void;
  stopCapture(): void;
  getMouseAndReset(): { dx: number; dy: number };
  isCaptureActive(): boolean;
}

export default TurboModuleRegistry.getEnforcing<Spec>('RNNSEvent');
```

---

## C++ Layer

The C++ layer implements the TurboModule spec class generated by codegen. It owns:

- TurboModule method implementations
- Callback storage (`std::function`)
- Delegation to Objective-C++ for platform APIs

The C++ layer MUST NOT:

- Import AppKit
- Import Foundation
- Reference NSWindow or NSEvent
- Reference Objective-C types

### Files

```text
cpp/
    RNNSEventModule.h
    RNNSEventModule.cpp
    MouseState.h
    MouseState.cpp
```

### Module Class

```cpp
#pragma once
#include "RNNSEventModuleSpec.h"  // codegen-generated
#include "MouseState.h"

namespace facebook::react {

class RNNSEventModule : public RNNSEventModuleSpec {
public:
  RNNSEventModule(std::shared_ptr<CallInvoker> jsInvoker);

  void registerEventCallback(std::function<void(folly::dynamic)> callback) override;
  void startCapture() override;
  void stopCapture() override;
  folly::dynamic getMouseAndReset() override;
  bool isCaptureActive() override;

private:
  std::function<void(folly::dynamic)> callback_;
};

} // namespace facebook::react
```

## Objective-C++ Layer (.mm)

The `.mm` layer owns:

- AppKit integration
- NSEvent monitoring
- NSWindow filtering
- Mouse capture state (CGAssociateMouseAndMouseCursorPosition, NSCursor)

The `.mm` layer is the **only** place AppKit types may appear.

### Allowed Types

```objc
NSWindow
NSEvent
NSApplication
NSCursor
```

### Files

```text
macos/
    RNNSEventModule.mm   ← platform implementation (method bodies that call AppKit)
```

# Data Flow

```
JS
↓
TurboModuleRegistry.getEnforcing<Spec>('RNNSEvent')
↓
C++ TurboModule (RNNSEventModule — state + callbacks)
↓
Objective-C++ (.mm — AppKit APIs)
↓
NSEvent monitor / NSWindow / NSCursor / CGAssociateMouseAndMouseCursorPosition
```

Callbacks flow back:

```
NSEvent monitor (main thread)
→ callback_ std::function (marshaled to JS thread via CallInvoker)
→ JS callback
```

# Event Monitoring

The module installs a local monitor:

```objc
_monitor =
[NSEvent addLocalMonitorForEventsMatchingMask:NSEventMaskAny
                                      handler:^NSEvent *(NSEvent *event) {

    ...

    return event;
}];
```

The monitor handle must be retained.

Cleanup must remove the monitor:

```objc
[NSEvent removeMonitor:_monitor];
```

---

# Window Filtering

Events are only processed when:

```objc
event.window == targetWindow
```

and:

```objc
targetWindow.isKeyWindow == YES
NSApp.isActive == YES
```

Events from all other windows must be ignored.

# Public API

## registerEventCallback

```ts
registerEventCallback(
  callback: ((event: NSEventEvent) => void) | null
): void;
```

Registers a single callback. The callback is stored as `std::function` in the C++ TurboModule.

Passing `null` unregisters the callback.

---

## startCapture

```ts
startCapture(): void;
```

Behavior (in .mm):

```objc
CGAssociateMouseAndMouseCursorPosition(false);
[NSCursor hide];
```

Must reset accumulators:

```cpp
mouseState.dx.store(0, std::memory_order_relaxed);
mouseState.dy.store(0, std::memory_order_relaxed);
```

Capture mode becomes active.

---

## stopCapture

```ts
stopCapture(): void;
```

Behavior (in .mm):

```objc
CGAssociateMouseAndMouseCursorPosition(true);
[NSCursor unhide];
```

Must reset accumulators:

```cpp
mouseState.dx.store(0, std::memory_order_relaxed);
mouseState.dy.store(0, std::memory_order_relaxed);
```

Capture mode becomes inactive.

---

## getMouseAndReset

```ts
getMouseAndReset(): {
  dx: number;
  dy: number;
};
```

Returns accumulated mouse movement and atomically resets the counters.

Intended to be called from:

```ts
requestAnimationFrame(...)
```

or another application-controlled update loop.

---

## isCaptureActive

```ts
isCaptureActive(): boolean;
```

Returns whether capture mode is active.

# Mouse State

## Definition

```cpp
struct MouseState {
    std::atomic<int64_t> dx{0};
    std::atomic<int64_t> dy{0};
};
```

Single shared instance:

```cpp
extern MouseState mouseState;
```

---

# Mouse Accumulation

## Capture Disabled

Mouse movement is ignored.

No JavaScript events emitted.

No accumulation performed.

---

## Capture Enabled

The following event types contribute to accumulation:

```text
NSEventTypeMouseMoved
NSEventTypeLeftMouseDragged
NSEventTypeRightMouseDragged
NSEventTypeOtherMouseDragged
```

Implementation:

```cpp
mouseState.dx.fetch_add(
    static_cast<int64_t>(event.deltaX),
    std::memory_order_relaxed);

mouseState.dy.fetch_add(
    static_cast<int64_t>(event.deltaY),
    std::memory_order_relaxed);
```

Mouse movement events must never be emitted to JavaScript.

JavaScript obtains movement exclusively through:

```ts
getMouseAndReset()
```

---

# Atomic Reset

Implementation:

```cpp
struct MouseDelta {
    int64_t dx;
    int64_t dy;
};

MouseDelta getMouseAndReset() {
    return {
        mouseState.dx.exchange(
            0,
            std::memory_order_acq_rel),
        mouseState.dy.exchange(
            0,
            std::memory_order_acq_rel),
    };
}
```

Requirements:

- Lock-free
- No mutexes
- No lost input
- No race conditions

# Threading Model

## Main Thread

Owns:

- NSEvent monitor
- AppKit APIs
- Event collection
- Mouse accumulation

All AppKit access must occur on the main thread.

---

## JavaScript Thread

Owns:

- Callback execution (invoked via CallInvoker)
- getMouseAndReset calls
- All TurboModule method calls from JS

---

## Callback Marshaling

When the NSEvent monitor fires on the main thread and needs to invoke the JS callback:

```cpp
jsInvoker_->invokeAsync([cb = callback_, event = std::move(eventData)]() {
  cb(event);
});
```

This ensures the callback executes on the JS thread.

---

## Synchronization

Shared mutable state is limited to:

```cpp
MouseState
```

Synchronization is performed exclusively through:

```cpp
std::atomic<int64_t>
```

No mutexes.

No locks.

No condition variables.

---

# Registration

The module uses codegen + auto-linking. No manual registration is required.

The TurboModule is accessed via:

```typescript
TurboModuleRegistry.getEnforcing<Spec>('RNNSEvent');
```

The system automatically wires the C++ class factory to the JS registry.

---

# Design Constraints

## Required

- Window-scoped input only
- Active window only
- Active application only
- No global hooks
- No CGEventTap
- No Input Monitoring permission
- No Accessibility permission
- No mouse move event spam to JavaScript
- Mouse movement retrieved through polling
- TurboModule architecture (not JSI host object)

---

## Not Supported

- Background input capture
- Global hotkeys
- Input remapping
- Multi-process input interception
- System-wide monitoring

---

# Summary

`react-native-nsevent` provides:

- Keyboard events
- Mouse button events
- Scroll events
- Relative mouse capture
- Atomic mouse delta accumulation
- Frame-synchronized polling through `getMouseAndReset()`

The implementation is based on:

- TurboModule with codegen-generated spec
- `TurboModuleRegistry` for module access
- `std::function` callbacks passed between C++ and JS
- `CallInvoker` for JS thread marshaling
- `NSEvent addLocalMonitorForEventsMatchingMask`
- `NSWindow` filtering
- `std::atomic<int64_t>`
- No CGEventTap
- No manual JSI host object installation
