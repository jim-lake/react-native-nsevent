# TurboModules (React Native) — Setup, Registration, and C++ ↔ Objective-C++ Bridge

This is a compact reference for React Native TurboModules covering setup, registration, boilerplate, and how to pass callbacks between C++ and Objective-C++ (.mm).

---

## 1. What a TurboModule is

A TurboModule is split into:

- JS (typed interface + TurboModuleRegistry)
- C++ (core implementation, state, logic, callbacks)
- Objective-C++ (.mm) (platform APIs like NSWindow, NSEvent, UIKit/AppKit)

Data flow:

JS → JSI → C++ TurboModule → Objective-C++ → native OS APIs  
and back via callbacks.

---

## 2. Typical file layout

/ios/MyModule/MyModule.mm        ← ObjC++ implementation  
/ios/MyModule/MyModule.h         ← C++ class  
/ios/MyModule/MyModuleSpec.h     ← generated by codegen  
/cpp/MyModule.cpp                ← optional shared logic  
/js/MyModule.ts                  ← JS interface  

---

## 3. JS interface (TypeScript spec)

import type { TurboModule } from 'react-native';
import { TurboModuleRegistry } from 'react-native';

export interface Spec extends TurboModule {
  add(a: number, b: number): number;
  setCallback(cb: (value: number) => void): void;
}

export default TurboModuleRegistry.getEnforcing<Spec>('MyModule');

---

## 4. C++ TurboModule implementation

// MyModule.h
#pragma once
#include "MyModuleSpec.h"

namespace facebook::react {

class MyModule : public MyModuleSpec {
public:
  MyModule(std::shared_ptr<CallInvoker> jsInvoker);

  int64_t add(double a, double b) override;
  void setCallback(std::function<void(double)> cb) override;

private:
  std::function<void(double)> callback_;
};

} // namespace facebook::react


// MyModule.cpp
#include "MyModule.h"

namespace facebook::react {

MyModule::MyModule(std::shared_ptr<CallInvoker> jsInvoker)
  : MyModuleSpec(std::move(jsInvoker)) {}

int64_t MyModule::add(double a, double b) {
  return (int64_t)(a + b);
}

void MyModule::setCallback(std::function<void(double)> cb) {
  callback_ = std::move(cb);

  if (callback_) {
    callback_(42.0);
  }
}

} // namespace facebook::react

## 5. Objective-C++ (.mm) integration layer

This is where Apple APIs live.

#import "MyModule.h"

using namespace facebook::react;

MyModule::MyModule(std::shared_ptr<CallInvoker> jsInvoker)
  : MyModuleSpec(std::move(jsInvoker)) {}

void MyModule::setCallback(std::function<void(double)> cb) {
  callback_ = std::move(cb);

  dispatch_async(dispatch_get_main_queue(), ^{
    callback_(100.0);
  });
}

## 6. Registration model (important)

### New architecture (TurboModules + codegen)

You typically DO NOT manually register modules.

Codegen generates bindings like:

RCT_EXTERN_MODULE(MyModule, NSObject)

and the system automatically wires:

- TurboModuleRegistry
- C++ class factory
- Objective-C++ implementation

---

### Old/manual fallback (rare)

[self.bridge moduleForClass:[MyModule class]];

But modern RN uses auto-linking + codegen.

---

## 7. C++ ↔ Objective-C++ calling

Because .mm is Objective-C++ you can freely mix:

### C++ → ObjC

dispatch_async(dispatch_get_main_queue(), ^{
  NSLog(@"C++ calling ObjC runtime");
});

NSString *msg = @"hello";
NSLog(@"%@", msg);

---

### ObjC → C++

self->callback_(123.0);

or:

MyModule* module = this;
module->add(1, 2);

---

## 8. Storing and invoking function pointers

You can safely store JS callbacks as:

std::function<void(double)> callback_;

Invoke anytime:

void MyModule::trigger() {
  if (callback_) {
    callback_(777.0);
  }
}

---

## 9. Async safe pattern

Always ensure lifetime safety:

std::weak_ptr<MyModule> weakSelf = shared_from_this();

dispatch_async(dispatch_get_main_queue(), ^{
  if (auto self = weakSelf.lock()) {
    if (self->callback_) {
      self->callback_(999.0);
    }
  }
});

---

## 10. Calling back into JS thread safely

jsInvoker_->invokeAsync([cb = std::move(callback_)]() {
  cb(123);
});

---

## 11. Threading rules

- JS thread: via CallInvoker only
- UI APIs: main thread only (AppKit/UIKit)
- C++: any thread, but you must marshal correctly

---

## 12. Mental model

JS  
↓  
TurboModuleRegistry  
↓  
C++ TurboModule (state + logic + callbacks)  
↓  
Objective-C++ (.mm system APIs)  
↓  
dispatch_async / NSEvent / NSWindow  

---

## 13. Key idea summary

- JS only sees typed API
- C++ owns logic + callbacks
- ObjC++ is just platform glue
- No manual registration in modern RN
- Callbacks are std::function passed across layers
- .mm is where OS integration actually happens
